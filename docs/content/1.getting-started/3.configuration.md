All options live under the `atproto` key in `nuxt.config.ts`. They are copied to `runtimeConfig.public.atproto` at build time.

## Full example

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['nuxt-atproto'],
  atproto: {
    serviceEndpoint: {
      private: 'https://bsky.social',
      public: 'https://public.api.bsky.app',
    },
    oauth: {
      writeClientMetadata: true,
      clientMetadata: {
        remote: '',
        local: {
          client_id: 'https://your-app.example/client-metadata.json',
          client_name: 'My App',
          client_uri: 'https://your-app.example',
          logo_uri: 'https://your-app.example/logo.png',
          tos_uri: 'https://your-app.example/tos',
          policy_uri: 'https://your-app.example/privacy',
          redirect_uris: ['https://your-app.example'],
          scope: 'atproto',
          grant_types: ['authorization_code', 'refresh_token'],
          response_types: ['code'],
          token_endpoint_auth_method: 'none',
          application_type: 'web',
          dpop_bound_access_tokens: true,
        },
      },
      signInOptions: {
        state: '',
        prompt: 'login',
        scope: 'atproto',
        ui_locales: 'en',
      },
    },
    debug: true,
  },
})
```

## Options reference

### `serviceEndpoint`

| Key | Default | Description |
|-----|---------|-------------|
| `private` | `https://bsky.social` | PDS used as handle resolver and default sign-in target |
| `public` | `https://public.api.bsky.app` | Public API for unauthenticated `AtpAgent` requests |

### `oauth.writeClientMetadata`

| Key | Default | Description |
|-----|---------|-------------|
| `writeClientMetadata` | `false` | When `true`, writes `public/client-metadata.json` from `oauth.clientMetadata.local` during module setup (if `public/` is writable). |

### `oauth.clientMetadata`

| Key | Description |
|-----|-------------|
| `remote` | URL of a hosted `client-metadata.json`. When set, the module loads metadata from this URL instead of `local`. |
| `local` | Inline OAuth client metadata used by the client plugin (and optionally written to disk when `writeClientMetadata` is `true`). |

::callout{type="warning"}
**Production checklist:** `client_id` must match the canonical URL of your metadata file. `redirect_uris` must include every origin users sign in from (production, preview, localhost if needed).
::

### `oauth.signInOptions`

Passed to `signInRedirect` / `authorize`. Defaults:

- `prompt`: `'login'`
- `scope`: `'atproto'`
- `ui_locales`: `'en'`

### `debug`

When `true`, logs OAuth init and sign-out messages to the console. **Session hooks fire regardless of this flag.**

## Local `client-metadata.json`

To generate the file at build/dev startup, set `oauth.writeClientMetadata: true` and fill in `oauth.clientMetadata.local`. The [playground `nuxt.config.ts`](https://github.com/dxlliv/nuxt-atproto/blob/main/playground/nuxt.config.ts) enables this for local OAuth.

If you host metadata yourself (recommended for production), keep `writeClientMetadata` at the default `false` and set `client_id` to your hosted URL.

In **development**, the plugin may omit inline metadata so the OAuth client can resolve it from your dev server — ensure metadata is reachable at the URL used as `client_id`.

## Remote metadata

Set `remote` to your hosted metadata URL:

```ts
oauth: {
  clientMetadata: {
    remote: 'https://your-app.example/client-metadata.json',
  },
},
```

Use this when you prefer a single canonical file in production rather than a build-time write.
