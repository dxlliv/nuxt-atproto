## Overview

**nuxt-atproto** is a [Nuxt](https://nuxt.com) module that adds [AT Protocol](https://atproto.com) OAuth authentication to your app. It is aimed at [Bluesky](https://bsky.app) and other ATProto services that support the standard OAuth client flow.

The module handles:

- Initializing a browser OAuth client on the client
- Storing and exposing the active OAuth session reactively
- Providing cached [`@atproto/api`](https://www.npmjs.com/package/@atproto/api) agents for authenticated and public requests
- Emitting Nuxt hooks when sessions are created, restored, or deleted

## Client-first by design

Authentication runs **only in the browser**. The module registers a client-only plugin (`mode: 'client'`) because OAuth depends on `window`, redirects, and browser storage.

::callout{type="warning"}
**Not server-side OAuth.** There is no Nitro runtime or server-side session in this module. On the server, session composables return empty state and auth methods are no-ops. Wrap login UI in `<ClientOnly>` or use `ssr: false` for pages that depend on an active session.
::

## What the module does not do

- Server-side token validation or BFF proxies — see [Security](/advanced/security)
- Pre-built login UI components (you own the buttons and forms)
- Hosting `client-metadata.json` for you (host it yourself, or opt in to `oauth.writeClientMetadata` to generate `public/client-metadata.json` at build time)

## Recommended composables

| Composable | Responsibility |
|------------|----------------|
| `useAtprotoSession()` | Reactive `session`, `isLogged`, `status` |
| `useAtprotoAuth()` | `signIn`, `signInWithHandle`, `signOut`, `restore` |
| `useAtprotoAgent(scope)` | Cached `Agent` / `AtpAgent` for API calls |

Legacy `useAtproto()` and `useAgent()` remain as thin wrappers but are deprecated — see [Migration](/advanced/migration).

This module covers **authentication and agents only**. Application features (feeds, posts, notifications) use [`@atproto/api`](https://www.npmjs.com/package/@atproto/api) directly after login.
