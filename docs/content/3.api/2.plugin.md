Client-only plugin (`mode: 'client'`). Not available during SSR.

## `AtprotoContext`

```ts
interface AtprotoContext {
  client: BrowserOAuthClient
  session: Ref<OAuthSession | undefined>
  status: Ref<AtprotoSessionStatus>
}
```

| Property | Description |
|----------|-------------|
| `client` | `@atproto/oauth-client-browser` instance |
| `session` | Reactive OAuth session; use `session.sub` as DID |
| `status` | `'initializing'`, `'authenticated'`, or `'anonymous'` at runtime |

## Access

```ts
const { $atproto } = useNuxtApp()
```

Prefer composables for app code. Use `$atproto` for low-level OAuth client access.

## Initialization sequence

1. Create `BrowserOAuthClient` — `clientMetadata.remote` URL or inline `local` (production; omitted in dev when empty)
2. `handleResolver` set to `serviceEndpoint.private`
3. `await client.init()` — OAuth callback or cold restore
4. `applyAtprotoSession(..., deferHooks: true)` on success; `status = 'anonymous'` on no session or error
5. `onDelete` → `nuxtApp.hooks.callHook('atproto:sessionDeleted', sub)`

Init errors are logged; `status` becomes `'anonymous'`.

See [Production patterns](/guide/production-patterns) for hook registration.
