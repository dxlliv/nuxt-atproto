Configure via `atproto` in `nuxt.config.ts`. Type: `AtprotoNuxtOptions`.

For setup walkthrough, production checklist, and scope notes, see [Configuration](/getting-started/configuration).

## Defaults

```ts
{
  serviceEndpoint: {
    private: 'https://bsky.social',
    public: 'https://public.api.bsky.app',
  },
  oauth: {
    writeClientMetadata: false,
    clientMetadata: {
      remote: '',
      local: {
        client_id: '',
        client_name: '',
        client_uri: '',
        logo_uri: '',
        tos_uri: '',
        policy_uri: '',
        redirect_uris: [],
        scope: 'atproto',
        grant_types: [],
        response_types: [],
        token_endpoint_auth_method: 'none',
        application_type: 'web',
        dpop_bound_access_tokens: true,
      },
    },
    signInOptions: {
      state: '',
      prompt: 'login',
      scope: 'atproto',
      ui_locales: 'en',
    },
  },
  debug: true,
}
```

## Fields

### `serviceEndpoint`

| Field | Type | Default |
|-------|------|---------|
| `private` | `string` | `https://bsky.social` |
| `public` | `string` | `https://public.api.bsky.app` |

### `oauth.writeClientMetadata`

| Field | Type | Default |
|-------|------|---------|
| `writeClientMetadata` | `boolean` | `false` |

### `oauth.clientMetadata`

| Field | Type |
|-------|------|
| `remote` | `string` — hosted metadata URL |
| `local` | inline metadata object |

### `oauth.signInOptions` (`AtprotoSignInOptions`)

| Field | Type | Default |
|-------|------|---------|
| `state` | `string` | `''` |
| `prompt` | `'login' \| 'consent'` | `'login'` |
| `scope` | `string` | `'atproto'` |
| `ui_locales` | `string` | `'en'` |

### `debug`

| Type | Default |
|------|---------|
| `boolean` | `true` |

Logs OAuth init/sign-out only. Does not gate hooks.

## Runtime config

Copied to `runtimeConfig.public.atproto` at build time. Read in app code via `useRuntimeConfig().public.atproto`.
