All hooks receive the account **DID** (`session.sub`) as a `string`.

| Hook | When | Deferred to `app:mounted` |
|------|------|---------------------------|
| `atproto:sessionCreated` | OAuth redirect completed (new login) | yes |
| `atproto:sessionRestored` | Last session restored on load, or `restore(did)` | init: yes; `restore()`: no |
| `atproto:sessionDeleted` | OAuth client `onDelete` (revoke, expiry, etc.) | no |

::callout{type="info"}
`signOut()` does **not** emit `sessionDeleted` — `revoke()` triggers `onDelete` instead. See [Sessions and hooks](/guide/sessions-and-hooks).
::

## TypeScript

```ts
declare module '#app' {
  interface RuntimeNuxtHooks {
    'atproto:sessionCreated': (did: string) => HookResult
    'atproto:sessionRestored': (did: string) => HookResult
    'atproto:sessionDeleted': (did: string) => HookResult
  }
}
```

Register in a client plugin — see [Production patterns](/guide/production-patterns).
