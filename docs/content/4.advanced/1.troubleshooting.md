## Login redirect fails or loops

1. **`redirect_uris`** in client metadata must exactly match your app origin (scheme, host, port).
2. **`client_id`** must be the HTTPS URL where `client-metadata.json` is served.
3. In production, prefer hosting metadata at a stable URL and set `oauth.clientMetadata.remote`.
4. **Local dev:** use `http://127.0.0.1:<port>`, not `http://localhost:<port>`. Browsers and OAuth treat them as different origins; a mismatch between the URL you open and `redirect_uris` breaks sign-in.

## Sign-in appears to do nothing

`signIn()` and `signInWithHandle()` catch errors and log to the console — they do not throw. Open DevTools → **Console** and look for `Error during sign-in:`.

Common causes: unreachable metadata, wrong `redirect_uris`, or handle resolution failure. See [Errors and debugging](/guide/errors-and-debugging).

## `client-metadata.json` not found

- Set `oauth.writeClientMetadata: true` if you want the module to generate `public/client-metadata.json` from `oauth.clientMetadata.local`.
- Otherwise host metadata at a stable URL and set `oauth.clientMetadata.remote` or point `client_id` at your hosted file.
- Open `https://your-origin/client-metadata.json` in the browser to verify it is served.
- If generation is enabled but the file is missing, check that `public/` is writable.

## Hooks never run

- Hooks fire on the **client** after `app:mounted` (create/restore).
- They are **not** gated by `debug: false` — if hooks are silent, verify listeners are registered in a client plugin or component.
- Ensure you are not only testing SSR output; OAuth does not run on the server.

## `Not authenticated` from agent

- Call `useAtprotoAgent('authenticated')` only when `useAtprotoSession().isLogged` is true and `status !== 'initializing'`.
- After `signOut()`, the authenticated agent cache is cleared — fetch again only after a new login or `restore()`.

## OAuth works in dev but not production

- Production uses inline `clientMetadata.local` (not omitted like dev). Ensure `local` is complete and deployed metadata URL matches `client_id`.
- Verify HTTPS everywhere; OAuth clients typically require secure redirect URIs.

## `useAtprotoAgent` throws on server

Expected. Move agent calls to client-only code paths (`onMounted`, `watch`, hooks, `<ClientOnly>`).

## Session deleted unexpectedly

`atproto:sessionDeleted` fires when the OAuth client removes a session. Check token expiry, revoke from another device, or `onDelete` causes in `@atproto/oauth-client-browser`.

## DevTools checklist

| Tab | Check |
|-----|-------|
| Console | OAuth init logs (`debug: true`), sign-in errors |
| Network | `client-metadata.json` returns 200; OAuth redirects succeed |
| Application | Browser storage entries if sessions persist incorrectly |
