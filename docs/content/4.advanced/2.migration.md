## Overview

The current API (package **0.1.0+**) uses focused composables and client-only guarantees. Legacy `useAtproto()` and `useAgent()` still work but are deprecated.

## Composables

| Before | After |
|--------|-------|
| `useAtproto()` for everything | `useAtprotoSession()` + `useAtprotoAuth()` |
| `useAtproto().isLogged()` | `useAtprotoSession().isLogged` (computed ref) |
| `useAtproto().getSession()` | `useAtprotoSession().session` |
| `useAgent('private')` | `useAtprotoAgent('authenticated')` |
| `useAgent('public')` | `useAtprotoAgent('public')` |

## Sign-in with handle

**Before** (optional handle + prompt):

```ts
await useAtproto().signInWithHandle()
await useAtproto().signInWithHandle('alice.bsky.social')
```

**After** (handle required):

```ts
const { signInWithHandle } = useAtprotoAuth()
await signInWithHandle('alice.bsky.social')
```

The deprecated `useAtproto()` wrapper still prompts when handle is omitted.

## Removed / never existed

Do not use these from older README drafts:

- `atproto.agent.account` — not part of the module
- SSR-friendly OAuth on the server — use client-only patterns instead

## Session hooks and `debug`

Previously, some examples assumed hooks only ran when `debug: true`. Hooks **always** run now; only log output depends on `debug`.

## `writeClientMetadata`

Client metadata is **not** written to `public/client-metadata.json` unless you set `oauth.writeClientMetadata: true`. Host metadata yourself in production, or enable the flag for local/dev generation (as in the playground).

## Identity helpers

`resolveActorDid` and `resolveActorServiceEndpoint` are no longer auto-imported from the module. Copy the pattern from [`playground/utils/resolveActor.ts`](https://github.com/dxlliv/nuxt-atproto/blob/main/playground/utils/resolveActor.ts) if you need them.

## Agent instances

If you called `useAgent()` in every component, upgrade to `useAtprotoAgent()` — caching is built in. No manual singleton needed.

## TypeScript

Add or update types by depending on `nuxt-atproto`; `$atproto` is augmented on `NuxtApp` when types are generated (`nuxt prepare`).

## Step-by-step migration

1. Replace `useAtproto()` with `useAtprotoSession` + `useAtprotoAuth` in login pages.
2. Replace `useAgent('private'|'public')` with `useAtprotoAgent('authenticated'|'public')`.
3. Wrap OAuth UI in `<ClientOnly>` if you relied on SSR rendering logged-in state.
4. Pass explicit handles to `signInWithHandle`.
5. Run `pnpm run test:types` in your app after upgrading.
