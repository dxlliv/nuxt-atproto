## `useAtprotoSession()`

```ts
const { session, isLogged, status } = useAtprotoSession()
```

| Property | Type | Description |
|----------|------|-------------|
| `session` | `Ref<OAuthSession \| undefined>` | Active OAuth session from the plugin |
| `isLogged` | `ComputedRef<boolean>` | `true` when `session` is set |
| `status` | `Ref<AtprotoSessionStatus>` | Plugin initialization and auth state |

### Session status values

| Status | Meaning |
|--------|---------|
| `initializing` | OAuth client is starting (`client.init()` in progress) |
| `authenticated` | A session is active after init or login |
| `anonymous` | No session (signed out, never logged in, or init failed) |

::callout{type="info"}
While `status === 'initializing'`, show a loading state instead of the logged-out UI to avoid a flash before a restored session appears.
::

On the **server**, `session` is always `undefined`, `isLogged` is `false`, and `status` is `'anonymous'`.

## Reading the session

```vue
<script setup lang="ts">
const { session, isLogged } = useAtprotoSession()
</script>

<template>
  <p v-if="isLogged">
    DID: {{ session?.sub }}
  </p>
</template>
```

The session object comes from `@atproto/oauth-client-browser`. Use `session.sub` as the account DID in most cases.

## Nuxt hooks

Register listeners with `useNuxtApp().hook()` — prefer a [client plugin](/guide/production-patterns#register-hooks-in-a-client-plugin) so hooks are set up once:

```ts
const { hook } = useNuxtApp()

hook('atproto:sessionCreated', (did: string) => {
  // User finished OAuth redirect (new login)
})

hook('atproto:sessionRestored', (did: string) => {
  // Previous session restored on page load
})

hook('atproto:sessionDeleted', (did: string) => {
  // Session removed (revoke, expiry, or OAuth client onDelete)
})
```

::callout{type="info"}
`atproto:sessionDeleted` is emitted from the OAuth client `onDelete` callback (for example after `revoke()`). `signOut()` clears session state via the module lifecycle but does **not** emit this hook again — avoid duplicate cleanup.
::

Create/restore hooks are scheduled on `app:mounted` so the DOM is ready for navigation or data fetching.

::callout{type="info"}
Hooks run whether or not `atproto.debug` is enabled. The `debug` flag only controls `console` output.
::

## Example: load profile after login

```ts
const { hook } = useNuxtApp()

async function loadProfile(did: string) {
  const agent = useAtprotoAgent('public')
  const { data } = await agent.getProfile({ actor: did })
  return data
}

hook('atproto:sessionCreated', async (did) => {
  const profile = await loadProfile(did)
  // persist or display profile
})

hook('atproto:sessionRestored', async (did) => {
  const profile = await loadProfile(did)
  // same handling as created
})
```

See [Multi-account sessions](/guide/multi-account) and the reference implementation in [`usePlaygroundSession`](https://github.com/dxlliv/nuxt-atproto/blob/main/playground/composables/usePlaygroundSession.ts) and [`PlaygroundDemo.vue`](https://github.com/dxlliv/nuxt-atproto/blob/main/playground/components/PlaygroundDemo.vue).
