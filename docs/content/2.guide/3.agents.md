## `useAtprotoAgent(scope)`

Returns an [`@atproto/api`](https://www.npmjs.com/package/@atproto/api) client. Instances are **cached per scope** in the app so ten components calling the same scope share one agent.

```ts
const agent = useAtprotoAgent('authenticated')
await agent.getTimeline()
```

::callout{type="warning"}
**Client only.** Calling `useAtprotoAgent` during SSR throws an error. Use it inside `onMounted`, `watchEffect`, event handlers, or hooks that run on the client.
::

## Scopes

| Scope | Returns | Use when |
|-------|---------|----------|
| `'authenticated'` | `Agent` bound to OAuth session | User-specific APIs (timeline, post, likes) |
| `'public'` | `AtpAgent` on `serviceEndpoint.public` | Profiles, public data without login |
| Custom URL string | `AtpAgent` on that service | Other PDS or AppView endpoints |

```ts
// Authenticated (requires active session)
const privateAgent = useAtprotoAgent('authenticated')

// Public Bluesky AppView
const publicAgent = useAtprotoAgent('public')

// Custom service
const customAgent = useAtprotoAgent('https://example.com')
```

Throws `Not authenticated` if you request `'authenticated'` without an active session.

## Caching behavior

- **Public / custom URL:** one `AtpAgent` per endpoint URL for the lifetime of the app (until full reload).
- **Authenticated:** one `Agent` per session DID (`session.sub`). Recreated when the session changes.

Cache invalidation happens automatically when you:

- call `signOut()` from `useAtprotoAuth`
- call `restore(did)` to switch accounts

## Optional custom `fetch`

Pass a custom fetch implementation as the second argument (same as `@atproto/api`):

```ts
const agent = useAtprotoAgent('public', myFetch)
```

## Deprecated `useAgent()`

`useAgent('private')` maps to the same cache as `useAtprotoAgent('authenticated')`. Prefer the new name in new code.

```ts
// Before
useAgent('private')

// After
useAtprotoAgent('authenticated')
```

## Performance tip

If many components fetch ATProto data, call `useAtprotoAgent` in each component without worry — you will not create duplicate agents for the same scope. Fetch in parallel from the shared instance or use your own data layer (`useAsyncData`, Pinia, etc.) on top.
