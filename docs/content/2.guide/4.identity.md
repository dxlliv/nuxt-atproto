The module does **not** ship identity helpers. For handle → DID → PDS flows, add utilities in your app (the [playground](https://github.com/dxlliv/nuxt-atproto/blob/main/playground/utils/resolveActor.ts) includes a reference implementation).

## Example utilities

```ts [utils/resolveActor.ts]
export async function resolveActorDid(handle: string): Promise<string> {
  const agent = useAtprotoAgent('public')

  const { data } = await agent.com.atproto.identity.resolveHandle({ handle })
  return data.did
}

export async function resolveActorServiceEndpoint(did: string): Promise<string> {
  const response = await fetch(`https://plc.directory/${did}`)

  if (!response.ok) {
    throw new Error('Failed to fetch profile service endpoint')
  }

  const data = await response.json()
  return data.service[0].serviceEndpoint
}
```

Both functions are **client-only** (`useAtprotoAgent` throws on the server).

## Handle to PDS URL

```ts
const handle = 'alice.bsky.social'
const did = await resolveActorDid(handle)
const pds = await resolveActorServiceEndpoint(did)

console.log(`${handle} → ${did} → ${pds}`)
```

## When agents are enough

For most Bluesky AppView operations, `useAtprotoAgent('public')` plus `com.atproto.identity.resolveHandle` is enough — you do not need separate helpers unless you want a reusable onboarding flow.
