## Two ways to sign in

### PDS redirect (`signIn`)

Starts the standard OAuth flow. The user picks their account on the PDS login UI.

```ts
const { signIn } = useAtprotoAuth()

await signIn()
// optional: custom PDS and sign-in options
await signIn('https://bsky.social', { scope: 'atproto', prompt: 'login' })
```

Under the hood this calls `$atproto.client.signInRedirect()` with `serviceEndpoint` (default: `atproto.serviceEndpoint.private`) and `oauth.signInOptions` from your config.

::callout{type="info"}
`signIn` and `signInWithHandle` catch errors and log to the console — they do not throw to callers. See [Errors and debugging](/guide/errors-and-debugging) if sign-in appears to do nothing.
::

### Handle-based (`signInWithHandle`)

Builds an authorization URL for a specific handle and navigates to it.

```ts
const { signInWithHandle } = useAtprotoAuth()

await signInWithHandle('alice.bsky.social')
```

::callout{type="warning"}
**Handle is required.** `signInWithHandle(handle: string)` must receive a non-empty handle. Build your own input UI — the module does not ship a login form.
::

## OAuth callback

After the user approves access, the provider redirects back to your app. The client plugin runs `BrowserOAuthClient.init()` on load, completes the callback, and sets `$atproto.session`.

If init returns a new login (`state` present), `atproto:sessionCreated` fires after `app:mounted`. If the last session was restored (`state` absent), `atproto:sessionRestored` fires instead.

## Sign out

```ts
const { signOut } = useAtprotoAuth()

await signOut()
```

This revokes the token when possible, clears the session ref, sets `status` to `'anonymous'`, and invalidates the cached authenticated agent.

## Restore another account

When you store multiple DIDs (e.g. in `localStorage`), switch sessions without a full redirect. See [Multi-account sessions](/guide/multi-account).

```ts
const { restore } = useAtprotoAuth()

const session = await restore('did:plc:your-did-here')
```

The private agent cache is invalidated so the next `useAtprotoAgent('authenticated')` uses the restored session.

## Legacy `useAtproto()`

`useAtproto().signIn()` and friends delegate to `useAtprotoAuth()`. The legacy wrapper still supports `signInWithHandle()` without a handle via `window.prompt` — prefer explicit handles in new code.
