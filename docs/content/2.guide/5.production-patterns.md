This page covers patterns you need beyond the [Usage](/getting-started/usage) quick start: loading states, profile fetch after login, client-only boundaries, and hook registration.

Reference implementation: [`usePlaygroundSession`](https://github.com/dxlliv/nuxt-atproto/blob/main/playground/composables/usePlaygroundSession.ts) and [`PlaygroundDemo.vue`](https://github.com/dxlliv/nuxt-atproto/blob/main/playground/components/PlaygroundDemo.vue).

## Handle `status === 'initializing'`

On first paint the OAuth client may still be restoring a session. Without a loading state, users briefly see the logged-out UI.

```vue
<script setup lang="ts">
const { isLogged, session, status } = useAtprotoSession()
const { signOut } = useAtprotoAuth()
</script>

<template>
  <ClientOnly>
    <p v-if="status === 'initializing'">
      Loading session…
    </p>
    <div v-else-if="isLogged">
      <p>Signed in as {{ session?.sub }}</p>
      <button type="button" @click="signOut()">Sign out</button>
    </div>
    <div v-else>
      <!-- login buttons -->
    </div>
  </ClientOnly>
</template>
```

## Client-only boundaries

| Approach | When to use |
|----------|-------------|
| `<ClientOnly>` | Wrap auth UI inside otherwise SSR pages |
| `ssr: false` on the route | Entire page depends on session (dashboard, settings) |
| `onMounted` / session hooks | Fetch authenticated data after hydration |

Do **not** call `useAtprotoAgent('authenticated')` during SSR — it throws. Public reads with `useAtprotoAgent('public')` are also client-only in practice because the plugin is not registered on the server.

## Recipe: login → profile → logout

### 1. Register hooks in a client plugin

Centralize post-login work so every page benefits:

```ts [plugins/atproto-session.client.ts]
export default defineNuxtPlugin(() => {
  const { hook } = useNuxtApp()

  async function onSession(did: string) {
    const agent = useAtprotoAgent('public')
    const { data } = await agent.getProfile({ actor: did })
    // store profile in your state layer (Pinia, useState, etc.)
  }

  hook('atproto:sessionCreated', onSession)
  hook('atproto:sessionRestored', onSession)
})
```

Create/restore hooks run after `app:mounted`. See [Sessions and hooks](/guide/sessions-and-hooks).

### 2. Fetch with loading and error states

```vue
<script setup lang="ts">
const { isLogged, status } = useAtprotoSession()
const profile = ref<{ handle: string, displayName?: string } | null>(null)
const loading = ref(false)
const error = ref<string | null>(null)

watch(
  () => [status.value, isLogged.value] as const,
  async ([currentStatus, loggedIn]) => {
    if (currentStatus === 'initializing' || !loggedIn) {
      profile.value = null
      return
    }

    loading.value = true
    error.value = null

    try {
      const agent = useAtprotoAgent('authenticated')
      const { data } = await agent.getProfile({ actor: 'me' })
      profile.value = { handle: data.handle, displayName: data.displayName }
    }
    catch (err) {
      error.value = err instanceof Error ? err.message : 'Failed to load profile'
    }
    finally {
      loading.value = false
    }
  },
  { immediate: true },
)
</script>

<template>
  <ClientOnly>
    <p v-if="loading">Loading profile…</p>
    <p v-else-if="error">{{ error }}</p>
    <p v-else-if="profile">
      {{ profile.displayName ?? profile.handle }}
    </p>
  </ClientOnly>
</template>
```

### 3. Sign out

```ts
const { signOut } = useAtprotoAuth()
await signOut()
```

Clears session state and invalidates the cached authenticated agent. Clear any app-local profile cache keyed by DID if needed (see [Multi-account sessions](/guide/multi-account)).

## Optional route middleware

Guard client routes that require auth:

```ts [middleware/auth.client.ts]
export default defineNuxtRouteMiddleware(() => {
  const { isLogged, status } = useAtprotoSession()

  if (status.value === 'initializing') {
    return
  }

  if (!isLogged.value) {
    return navigateTo('/login')
  }
})
```

Use `.client.ts` suffix so the middleware never runs on the server where session is always empty.
